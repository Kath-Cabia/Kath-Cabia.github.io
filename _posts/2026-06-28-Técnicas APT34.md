# **TÉCNICAS APT 34**

# **TÁCTICA 1: Reconnaissance**

## T1592: Gather Victim Host Information (Recopilar información sobre el anfitrión de la víctima)

**1. DESCRIPCIÓN:**
La técnica Gather Victim Host Information consiste en recopilar información sobre los equipos de la víctima antes de realizar un ataque. 

**¿Qué capacidad otorga al atacante?**
El atacante busca conocer datos como la dirección IP, el nombre del equipo, el sistema operativo, el idioma y otras características de su configuración. Esta información le permite identificar posibles vulnerabilidades y planificar un ataque más preciso.


**2. EJECUCIÓN:**
Para obtener información del equipo objetivo, el atacante puede seguir diferentes métodos, entre ellos:

- Escaneo activo (Active Scanning): utiliza herramientas como Nmap para identificar equipos, puertos abiertos y servicios disponibles.
- Phishing: engaña al usuario para que proporcione información del sistema o del dispositivo.
- Sitios web comprometidos: inserta contenido malicioso que recopila información de los visitantes.
- Fuentes públicas: consulta sitios web, redes sociales o bases de datos técnicas para obtener información expuesta por la organización.
- Análisis del User-Agent: examina los encabezados HTTP enviados por el navegador para identificar el sistema operativo, el navegador y su versión.

Para que esta técnica sea efectiva, la información debe estar disponible públicamente o el usuario debe interactuar con los recursos utilizados por el atacante.

**3. PRODUCTO:**
Si la técnica se ejecuta con éxito, el atacante obtiene:

-Dirección IP y nombre del equipo.
-Sistema operativo, idioma y configuración del dispositivo.
-Información sobre los servicios y aplicaciones en uso.
-Datos que le permiten seleccionar las herramientas o vulnerabilidades más adecuadas para comprometer el sistema en las siguientes etapas del ataque.

# **TÁCTICA 2: Resource Development**

## T1583: Acquire Infrastructure (Adquirir infraestructura)

**1. Descripción**

La técnica Acquire Infrastructure consiste en obtener la infraestructura necesaria para preparar un ataque antes de dirigirse a la víctima. 

El atacante adquiere recursos como servidores, dominios, servicios en la nube o proxies para alojar malware, controlar equipos comprometidos o comunicarse con ellos. Esto le permite ocultar mejor su identidad, hacer que el tráfico parezca legítimo y facilitar la ejecución del ataque.

**2. Ejecución**

Antes de iniciar el ataque, el atacante obtiene la infraestructura que utilizará durante la operación. Generalmente realiza las siguientes acciones:

- Contrata o alquila servidores físicos o en la nube para alojar malware o un servidor de comando y control (C2).
- Registra o compra dominios que aparenten ser legítimos y puedan utilizarse en campañas de phishing o distribución de malware.
- Utiliza servicios web de terceros para que el tráfico malicioso pase desapercibido.
- Adquiere proxies o proxies residenciales para ocultar la dirección IP real desde donde se realizan las conexiones.
- Compra o alquila botnets para ejecutar ataques desde equipos previamente comprometidos.
- Aprovecha periodos de prueba gratuitos ofrecidos por proveedores de servicios para reducir costos y cambiar rápidamente de infraestructura cuando sea necesario.

**Herramientas o servicios utilizados:**

- Proveedores de servidores en la nube (VPS o Cloud).
- Servicios de registro de dominios.
- Plataformas de alojamiento web.
- Servicios de proxy o VPN.
- Botnets disponibles para alquiler o compra.

**NOTA:** Esta técnica no requiere comandos específicos, ya que se basa en adquirir y configurar infraestructura antes de iniciar el ataque.

**3. Producto**

Al ejecutar esta técnica con éxito, el atacante obtiene:

- Infraestructura preparada para ejecutar el ataque.
- Servidores donde alojar malware o un servidor C2.
- Dominios y sitios web que aparentan ser legítimos.
- Proxies o direcciones IP que ayudan a ocultar su identidad.
- Una plataforma que puede modificarse, ampliarse o eliminarse rápidamente para dificultar su rastreo y mantener la operación activa.

# **TÁCTICA 3: Initial Access**

## T1566.002: Phishing: Spearphishing Link (Phishing: Enlace de phishing)

**1. Descripción**
La técnica Spearphishing Link consiste en enviar un correo electrónico dirigido a una persona u organización específica con un enlace malicioso. 

El objetivo es convencer a la víctima de hacer clic en el enlace para obtener acceso al sistema, robar información o instalar malware. A diferencia de otros ataques de phishing, aquí el archivo malicioso no se adjunta al correo, sino que se descarga o ejecuta después de que la víctima accede al enlace.

**2. Ejecución**
Para llevar a cabo esta técnica, el atacante generalmente sigue estos pasos:

- Recopila información sobre la víctima para personalizar el correo y hacerlo más creíble.
- Prepara un sitio web o enlace malicioso, que puede dirigir a una página falsa, descargar malware o solicitar credenciales.
- Envía el correo electrónico, haciéndose pasar por una persona, empresa o institución de confianza.
- Espera que la víctima haga clic en el enlace y siga las instrucciones mostradas en la página.
- Obtiene acceso al equipo o a la cuenta de la víctima mediante la descarga de malware, el robo de credenciales o la autorización de una aplicación maliciosa.

**Herramientas utilizadas:**

- Plataformas de envío de correos electrónicos.
- Sitios web falsos o comprometidos.
- Frameworks de phishing (por ejemplo, GoPhish para campañas de simulación).
- Servidores para alojar páginas de inicio de sesión falsas o distribuir malware.

**Condiciones necesarias:**

- La víctima debe recibir el correo.
- Debe confiar en el mensaje y hacer clic en el enlace.
- En algunos casos, debe ingresar sus credenciales o autorizar el acceso a una aplicación.

**3. Producto**
Si la técnica tiene éxito, el atacante puede obtener:

- Credenciales de acceso del usuario.
- Instalación de malware en el equipo de la víctima.
- Acceso inicial a la red o al sistema comprometido.
- Tokens de autenticación o permisos sobre aplicaciones en la nube.
- Información que le permita continuar con otras etapas del ataque, como el movimiento - lateral o el robo de datos.

# **TÁCTICA 4: Persistence**
## T1053.005: Scheduled Task/Job: Scheduled Task (Tarea/trabajo programado: Tarea programada)

**1. Descripción**
La técnica Scheduled Task consiste en utilizar el Programador de tareas de Windows para ejecutar un programa de forma automática en una fecha, hora o evento determinado. 

Un atacante puede aprovechar esta función para ejecutar código malicioso al iniciar el sistema, cuando un usuario inicia sesión o en intervalos periódicos. Esto le permite mantener el acceso al equipo (persistencia) y ejecutar acciones sin intervención del usuario.

**2. Ejecución**

Generalmente, el atacante realiza los siguientes pasos:

- Obtiene acceso al equipo mediante otra técnica, como phishing o la explotación de una vulnerabilidad.
- Crea una tarea programada para ejecutar un programa o script malicioso automáticamente.
- Define cuándo se ejecutará la tarea (al iniciar Windows, al iniciar sesión o en un horario específico).
- Oculta o modifica la tarea, si tiene privilegios suficientes, para dificultar su detección.
- Mantiene la persistencia, ya que el programa volverá a ejecutarse según la programación establecida.

**Herramientas utilizadas:**

- Programador de tareas de Windows (Task Scheduler).
- Símbolo del sistema (CMD).
- PowerShell.
- Windows Management Instrumentation (WMI).

**Comandos comunes:**

Crear una tarea programada:

    schtasks /create /tn "MiTarea" /tr "C:\archivo.exe" /sc onlogon

Ver las tareas programadas:

    schtasks /query

Ejecutar una tarea manualmente:

    schtasks /run /tn "MiTarea"

Eliminar una tarea:

    schtasks /delete /tn "MiTarea" /f

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Ejecución automática de programas o scripts maliciosos.
- Persistencia en el sistema, incluso después de reiniciar el equipo.
- Posibilidad de ejecutar procesos con privilegios elevados, como SYSTEM, si la tarea fue configurada con esos permisos.
- Mayor dificultad para detectar la actividad maliciosa, especialmente si la tarea ha sido ocultada o modificada.

# **TÁCTICA 5: Privilege Escalation**
## T1078.002: Valid Accounts: Domain Accounts (TCuentas válidas: Cuentas de dominio)

**1. Descripción**
La técnica Domain Accounts consiste en utilizar credenciales válidas de una cuenta de dominio para acceder a los recursos de una organización. 

Estas cuentas son administradas por Active Directory y pueden pertenecer a usuarios, administradores o servicios. Al utilizar credenciales legítimas, el atacante puede pasar desapercibido, ya que sus acciones pueden parecer las de un usuario autorizado.

**2. Ejecución**
Para aplicar esta técnica, el atacante normalmente sigue estos pasos:

- Obtiene las credenciales de una cuenta de dominio mediante phishing, robo de credenciales, reutilización de contraseñas o técnicas como el volcado de credenciales (Credential Dumping).
- Inicia sesión utilizando una cuenta válida en el dominio.
- Accede a recursos compartidos, servidores o equipos a los que la cuenta tiene permisos.
- Aprovecha los privilegios de la cuenta para moverse por la red, mantener el acceso o elevar privilegios si la cuenta pertenece a un administrador.

**Herramientas utilizadas:**

- Active Directory.
- PowerShell.
- Símbolo del sistema (CMD).
- Herramientas de administración de Windows.

**Comandos comunes:**

Ver el usuario actual:

    whoami

Ver los grupos a los que pertenece la cuenta:

    whoami /groups

Consultar los usuarios del dominio:

    net user /domain

Consultar los grupos del dominio:

    net group /domain

Ver los equipos del dominio:

    net view /domain

**Nota:** Estos comandos son herramientas legítimas de administración de Windows. En el contexto de MITRE ATT&CK, un atacante podría utilizarlos después de comprometer una cuenta de dominio para conocer los recursos disponibles y ampliar su acceso.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante puede obtener:

- Acceso legítimo a equipos y recursos del dominio.
- Persistencia mediante el uso continuo de una cuenta válida.
- Acceso a información compartida y servicios internos.
- Posibilidad de moverse lateralmente entre equipos del dominio.
- Escalamiento de privilegios si la cuenta comprometida pertenece a un administrador o tiene permisos elevados.

# **TÁCTICA 6: Privilege Escalation**

## T1068: Exploitation for Privilege Escalation (Explotación para la escalada de privilegios)

**1. Descripción**
La técnica Exploitation for Privilege Escalation consiste en aprovechar una vulnerabilidad presente en el sistema operativo, un servicio o un programa para obtener permisos más altos dentro del equipo. 

En muchos casos, un atacante inicia con una cuenta de usuario con permisos limitados y utiliza una falla del sistema para convertirse en administrador o incluso obtener privilegios de SYSTEM o root. Esto le permite acceder a recursos que normalmente estarían protegidos.

**2. Ejecución**
Para llevar a cabo esta técnica, el atacante generalmente sigue estos pasos:

- Obtiene acceso inicial al equipo mediante otra técnica, como phishing o la explotación de una vulnerabilidad.
- Identifica la versión del sistema operativo, el kernel y los programas instalados para buscar vulnerabilidades conocidas.
- Analiza los privilegios actuales del usuario y verifica si existe alguna falla que permita elevarlos.
- Aprovecha la vulnerabilidad para ejecutar código con permisos más altos.
- Obtiene privilegios elevados, lo que le permite realizar acciones restringidas para un usuario normal.

**Herramientas utilizadas:**

- Windows CMD o PowerShell.
- Herramientas de identificación de vulnerabilidades.
- Bases de datos de CVE.
- Frameworks de pruebas de seguridad, como Metasploit (cuando existe un módulo para la vulnerabilidad).

**Comandos comunes de reconocimiento:**

Ver el usuario actual:

    whoami

Ver los privilegios del usuario:

    whoami /priv

Consultar la versión de Windows:

    systeminfo

Mostrar información del sistema:

    hostname

**Nota:** Estos comandos no realizan la escalación de privilegios. Se utilizan para identificar el entorno y determinar si el sistema podría ser vulnerable a una falla conocida.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante puede obtener:

- Permisos de administrador o de SYSTEM (en Windows) o root (en Linux).
- Acceso a archivos y configuraciones protegidas.
- Capacidad para instalar, modificar o eliminar programas y servicios.
- Posibilidad de desactivar mecanismos de seguridad o evadir controles de acceso.
- Mayor control sobre el sistema para continuar con otras fases del ataque, como la persistencia, el movimiento lateral o la extracción de información.

# **TÁCTICA 7: Discovery**

## T1082: System Information Discovery (Descubrimiento de información del sistema)

**1. Descripción**
La técnica System Information Discovery consiste en recopilar información sobre el sistema operativo y el hardware del equipo comprometido. 

El objetivo es conocer detalles como la versión del sistema, la arquitectura, los parches instalados, el nombre del equipo y otras características que ayuden a identificar posibles vulnerabilidades o definir el siguiente paso del ataque.

**2. Ejecución**
Una vez que el atacante obtiene acceso al equipo, comienza a recopilar información del sistema siguiendo estos pasos:

- Identifica el sistema operativo y su versión.
- Consulta la arquitectura del equipo (32 o 64 bits).
- Verifica los parches y actualizaciones instalados.
- Obtiene información del hardware, como el procesador o la memoria.
- Analiza los datos obtenidos para decidir qué herramientas o vulnerabilidades pueden utilizarse en las siguientes fases del ataque.

**Herramientas utilizadas:**

- Símbolo del sistema (CMD).
- PowerShell.
- Utilidad Systeminfo de Windows.
- Herramientas propias del sistema operativo.

**Comandos comunes:**

Mostrar información general del sistema:

    systeminfo

Ver el nombre del equipo:

    hostname

Ver el usuario que ha iniciado sesión:

    whoami

Consultar la versión de Windows:

    ver

Mostrar la arquitectura del sistema:

    wmic os get OSArchitecture

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Nombre del equipo y del usuario.
- Versión y arquitectura del sistema operativo.
- Información sobre parches y actualizaciones instaladas.
- Datos del hardware, como procesador y memoria.
- Información suficiente para identificar posibles vulnerabilidades y seleccionar las herramientas o técnicas más adecuadas para continuar con el ataque.

# **TÁCTICA 8: Discovery**
## T1033: System Owner/User Discovery (Descubrimiento del propietario/usuario del sistema)

**1. Descripción**
La técnica System Owner/User Discovery consiste en obtener información sobre los usuarios de un sistema. 

El atacante busca identificar quién está utilizando el equipo, qué cuentas existen y cuáles tienen mayores privilegios. Esta información le ayuda a decidir cómo continuar el ataque, qué cuentas pueden ser un objetivo y qué acciones puede realizar sin llamar la atención.

**2. Ejecución**
Una vez que el atacante obtiene acceso al equipo, puede realizar las siguientes acciones:

- Identificar el usuario actual que ha iniciado sesión.
- Consultar las cuentas de usuario existentes en el sistema.
- Verificar los grupos y privilegios asociados a cada usuario.
- Analizar la información obtenida para determinar si existen cuentas con permisos elevados o de administrador.

**Herramientas utilizadas:**

- Símbolo del sistema (CMD).
- PowerShell.
- Utilidades nativas de Windows y Linux.

**Comandos comunes (Windows):**

Ver el usuario actual:

    whoami

Mostrar el nombre del usuario actual:

    echo %USERNAME%

Listar las cuentas de usuario del equipo:

    net user

Ver los grupos a los que pertenece el usuario:

    whoami /groups

Consultar los usuarios del dominio (si el equipo pertenece a un dominio):

    net user /domain

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- El nombre del usuario que ha iniciado sesión.
- La lista de cuentas de usuario disponibles en el sistema.
- Información sobre grupos y privilegios de cada cuenta.
- Datos que le permiten identificar cuentas con permisos elevados.
- Información útil para planificar otras técnicas, como el movimiento lateral, la escalación de privilegios o el acceso a recursos restringidos.

# **TÁCTICA 9: Discovery**

## T1016: System Network Configuration Discovery (Descubrimiento de la configuración de la red del sistema)

**1. Descripción**
La técnica System Network Configuration Discovery consiste en recopilar información sobre la configuración de red de un equipo. 

El atacante busca conocer datos como la dirección IP, la dirección MAC, la puerta de enlace, los servidores DNS y las rutas de red. Con esta información puede entender cómo está conectada la máquina, identificar otros equipos de la red y planificar los siguientes pasos del ataque.

**2. Ejecución**
Una vez que obtiene acceso al sistema, el atacante suele realizar las siguientes acciones:

- Consultar la configuración de red del equipo.
- Identificar la dirección IP y la dirección MAC de las interfaces de red.
- Revisar la puerta de enlace y los servidores DNS configurados.
- Analizar la tabla de rutas para conocer cómo se comunica el equipo con otras redes.
- Utilizar la información obtenida para localizar otros dispositivos y preparar técnicas como el movimiento lateral o el reconocimiento de la red.

**Herramientas utilizadas:**

- Símbolo del sistema (CMD).
- PowerShell.
- Utilidades de administración de red de Windows.

**Comandos comunes (Windows):**

Mostrar la configuración completa de red:

    ipconfig /all

Mostrar la tabla ARP:

    arp -a

Ver la tabla de rutas del sistema:

    route print

Consultar información de NetBIOS:

    nbtstat -n

Mostrar las conexiones de red activas:

    netstat -ano

**Nota:** Estos comandos no modifican la configuración del equipo. Su función es recopilar información sobre la red para conocer el entorno y tomar decisiones sobre las siguientes etapas del ataque.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Dirección IP y dirección MAC del equipo.
- Información sobre la puerta de enlace y los servidores DNS.
- Configuración de las interfaces de red.
- Tabla de rutas y conexiones de red activas.
- Datos que le permiten comprender la estructura de la red, identificar otros equipos y planificar acciones posteriores, como el reconocimiento o el movimiento lateral.

# **TÁCTICA 10: Discovery**

## T1087: Account Discovery (Descubrimiento de cuentas)

**1. Descripción**
La técnica Account Discovery consiste en identificar las cuentas de usuario existentes en un sistema o en una red. 

El atacante busca conocer qué usuarios son válidos, cuáles tienen mayores privilegios y qué cuentas podrían ser utilizadas para acceder a otros recursos. Esta información le permite planificar mejor las siguientes etapas del ataque, como el robo de credenciales, el movimiento lateral o la escalación de privilegios.

**2. Ejecución**
Una vez que el atacante obtiene acceso al sistema, puede realizar las siguientes acciones para recopilar información sobre las cuentas:

- Consultar las cuentas de usuario registradas en el equipo.
- Identificar los grupos a los que pertenece cada usuario.
- Verificar si el equipo pertenece a un dominio, para conocer si existen cuentas administradas por Active Directory.
- Analizar los permisos de las cuentas encontradas e identificar aquellas con privilegios elevados.
- Utilizar la información obtenida para seleccionar cuentas que puedan ser objetivo de otras técnicas, como el acceso con credenciales válidas o el movimiento lateral.

**Herramientas utilizadas:**

- Símbolo del sistema (CMD).
- PowerShell.
- Active Directory (en entornos de dominio).
- Utilidades nativas de Windows.

**Comandos comunes (Windows):**

Listar las cuentas de usuario locales:

    net user

Consultar la información de un usuario específico:

    net user nombre_usuario

Ver los grupos locales del equipo:

    net localgroup

Listar los usuarios del dominio (si el equipo pertenece a un dominio):

    net user /domain

Mostrar el usuario actual:

    whoami

**Nota:** Estos comandos son herramientas de administración de Windows y no modifican el sistema. Su finalidad es recopilar información sobre las cuentas disponibles y sus permisos.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Lista de cuentas de usuario locales o de dominio.
- Información sobre grupos y permisos de cada cuenta.
- Identificación de cuentas con privilegios administrativos.
- Conocimiento de la estructura de usuarios del sistema o de la organización.
- Información que facilita ataques posteriores, como el uso de credenciales válidas, el movimiento lateral o la escalación de privilegios.

# **TÁCTICA 11: Discovery**

## T1069: Permission Groups Discovery (Descubrimiento de grupos de permisos)

**1. Descripción**
La técnica Permission Groups Discovery consiste en identificar los grupos de usuarios y los permisos configurados en un sistema o en un dominio. 

El atacante busca conocer qué grupos existen, qué usuarios pertenecen a ellos y cuáles tienen privilegios elevados. Esta información le permite identificar posibles objetivos para acceder a recursos importantes o aumentar sus privilegios dentro de la red.

**2. Ejecución**
Una vez que el atacante obtiene acceso al sistema, puede realizar las siguientes acciones:

- Consultar los grupos creados en el equipo o en el dominio.
- Identificar los usuarios que pertenecen a cada grupo.
- Verificar qué grupos tienen permisos administrativos o acceso a recursos críticos.
- Analizar la información obtenida para seleccionar cuentas con mayores privilegios y planificar las siguientes fases del ataque.

**Herramientas utilizadas:**

- Símbolo del sistema (CMD).
- PowerShell.
- Active Directory (en equipos unidos a un dominio).
- Utilidades nativas de Windows.

**Comandos comunes (Windows):**

Listar los grupos locales del equipo:

    net localgroup

Ver los miembros de un grupo local:

    net localgroup Administradores

Listar los grupos del dominio:

    net group /domain

Ver los miembros de un grupo del dominio:

    net group "Domain Admins" /domain

Mostrar los grupos a los que pertenece el usuario actual:

    whoami /groups

**Nota:** Estos comandos son herramientas legítimas de administración de Windows. En el contexto de MITRE ATT&CK, un atacante puede utilizarlos para conocer la estructura de permisos del sistema e identificar cuentas con privilegios elevados.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Lista de grupos locales o del dominio.
- Información sobre los usuarios que pertenecen a cada grupo.
- Identificación de grupos con privilegios administrativos o acceso a recursos críticos.
- Conocimiento de la estructura de permisos de la organización.
- Información útil para planificar otras técnicas, como la escalación de privilegios, el movimiento lateral o el uso de cuentas con permisos elevados.

# **TÁCTICA 12: Lateral Movement**

## T1021.001: Remote Services: Remote Desktop Protocol (RDP) (Servicios remotos: Protocolo de escritorio remoto)

**1. Descripción**
La técnica Remote Desktop Protocol (RDP) consiste en utilizar el protocolo de escritorio remoto para conectarse a otro equipo de la red mediante una cuenta de usuario válida. 

Una vez establecida la conexión, el atacante puede interactuar con el sistema como si estuviera sentado frente al equipo, lo que le permite ejecutar programas, acceder a archivos y continuar con otras acciones dentro de la red.

**2. Ejecución**
Para utilizar esta técnica, el atacante generalmente sigue estos pasos:

- Obtiene credenciales válidas de un usuario o administrador, por ejemplo, mediante phishing o robo de credenciales.
- Verifica que el servicio de Escritorio Remoto (RDP) esté habilitado en el equipo de destino.
- Establece la conexión remota utilizando las credenciales obtenidas.
- Accede al escritorio del sistema remoto y realiza las acciones permitidas por los privilegios de la cuenta utilizada.
- Aprovecha el acceso para administrar el equipo, desplazarse a otros sistemas o mantener la persistencia.

**Herramientas utilizadas:**

- Remote Desktop Connection (mstsc).
- Símbolo del sistema (CMD).
-PowerShell.
- Servicios de Escritorio Remoto de Windows (RDS).

**Comandos comunes (Windows):**

Abrir el cliente de Escritorio Remoto:

    mstsc

Verificar si el servicio de Escritorio Remoto está en ejecución:

    sc query TermService

Mostrar información de las sesiones activas:

    query user

Mostrar el usuario que inició sesión:

    whoami

Comprobar la conectividad con el puerto RDP (3389):

    netstat -an | find "3389"

**Nota:** Estos comandos son herramientas legítimas de administración de Windows. En el contexto de **MITRE ATT&CK**, un atacante puede utilizarlos para verificar el servicio RDP y acceder a un equipo remoto únicamente si ya dispone de credenciales válidas y el servicio está habilitado.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Acceso remoto e interactivo al escritorio del equipo.
- Capacidad para ejecutar programas y administrar el sistema según los permisos de la cuenta utilizada.
- Acceso a archivos, aplicaciones y recursos compartidos del equipo remoto.
- Posibilidad de desplazarse a otros equipos de la red (movimiento lateral).
- Un punto de apoyo para continuar con otras técnicas, como la persistencia, la escalación de privilegios o la extracción de información.

# **TÁCTICA 13: Collection**

## T1005: Data from Local System (Datos del sistema local)

**1. Descripción**
La técnica Data from Local System consiste en buscar y recopilar información almacenada en el equipo comprometido. 

El atacante intenta localizar archivos, documentos, bases de datos, archivos de configuración o cualquier otro dato que pueda ser de interés. El objetivo es identificar información valiosa antes de copiarla o extraerla del sistema.

**2. Ejecución**
Después de obtener acceso al equipo, el atacante suele realizar las siguientes acciones:

- Explora las unidades y carpetas para localizar archivos importantes.
- Busca documentos específicos, como archivos PDF, Word, Excel o bases de datos.
- Revisa archivos de configuración que puedan contener credenciales o información sensible.
- Identifica la ubicación de los datos de interés para recopilarlos posteriormente.
- Organiza la información encontrada para preparar su extracción o análisis.

**Herramientas utilizadas:**

- Símbolo del sistema (CMD).
- PowerShell.
- Explorador de archivos de Windows.
- Scripts de automatización para recopilar información.

**Comandos comunes (Windows):**

Mostrar el contenido de una carpeta:

    dir

Buscar todos los archivos PDF en el disco:

    dir C:\*.pdf /s

Buscar todos los archivos de Word:

    dir C:\*.docx /s

Buscar todos los archivos de Excel:

    dir C:\*.xlsx /s

Buscar archivos que contengan una palabra específica:

    findstr /S /I "contraseña" C:\*.txt

**Nota:** Estos comandos son herramientas legítimas de Windows para la administración y búsqueda de archivos. En el contexto de **MITRE ATT&CK**, un atacante puede utilizarlos para identificar información de interés antes de una posible extracción de datos.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Documentos, hojas de cálculo, presentaciones y otros archivos de interés.
- Archivos de configuración con información del sistema o de aplicaciones.
- Bases de datos u otros datos almacenados localmente.
- Información sensible, como credenciales, claves o datos personales, si están disponibles en el equipo.
- Un conjunto de archivos organizados y listos para ser analizados o extraídos en una etapa posterior del ataque.

# **TÁCTICA 14: Collection**

## T1074.001: Data Staged: Local Data Staging (Datos preparados: Preparación local de datos)

**1. Descripción**
La técnica Local Data Staging consiste en reunir y organizar los datos recopilados en una ubicación específica del equipo antes de extraerlos. 

En lugar de enviar cada archivo por separado, el atacante los agrupa en una carpeta o los combina en un solo archivo, lo que facilita su administración y una posible transferencia posterior. Esto también ayuda a reducir el tiempo necesario para recopilar la información durante las siguientes etapas del ataque.

**2. Ejecución**
Después de localizar los archivos de interés, el atacante normalmente realiza las siguientes acciones:

- Crea una carpeta donde almacenará temporalmente la información recopilada.
- Copia o mueve los archivos importantes hacia esa ubicación.
- Organiza los documentos por tipo o contenido para facilitar su revisión.
- Agrupa los archivos, si es necesario, utilizando un formato comprimido para simplificar su manejo.
- Prepara la información para su análisis o para una posible extracción en una etapa posterior.

**Herramientas utilizadas:**

- Símbolo del sistema (CMD).
- PowerShell.
- Explorador de archivos de Windows.
- Herramientas de compresión de archivos (por ejemplo, ZIP).

**Comandos comunes (Windows):**

Crear una carpeta para almacenar los archivos:

    mkdir C:\Temp\Datos

Copiar archivos a la carpeta creada:

    copy C:\Documentos\*.docx C:\Temp\Datos

Mover archivos a la carpeta temporal:

    move C:\Documentos\*.pdf C:\Temp\Datos

Mostrar el contenido de la carpeta:

    dir C:\Temp\Datos

Comprimir una carpeta mediante PowerShell:

    Compress-Archive -Path C:\Temp\Datos -DestinationPath C:\Temp\Datos.zip

**Nota:** Estos comandos son herramientas legítimas de administración de Windows. En el contexto de MITRE ATT&CK, un atacante puede utilizarlos para organizar la información recopilada antes de realizar otras acciones, como su análisis o una posible extracción.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Una carpeta centralizada con los archivos de interés.
- Documentos organizados y listos para su revisión.
- Archivos agrupados o comprimidos para facilitar su manejo.
- Una estructura de datos que simplifica las siguientes etapas del ataque, como el análisis o la exfiltración.

# **TÁCTICA 15: Command and Control**

## T1071.001: Application Layer Protocol: Web Protocols (Protocolo de capa de aplicación: Protocolos web)

**1. Descripción**
La técnica Application Layer Protocol: Web Protocols consiste en utilizar protocolos web, como HTTP y HTTPS, para intercambiar información entre un equipo comprometido y un servidor controlado por el atacante. Como estos protocolos son utilizados diariamente para navegar por Internet, el tráfico generado puede parecer normal y pasar desapercibido entre las comunicaciones legítimas de la red.

Esta técnica permite al atacante enviar instrucciones al equipo comprometido, recibir información del sistema o mantener la comunicación con el dispositivo, reduciendo las posibilidades de ser detectado.

**2. Ejecución**
Después de comprometer un equipo, el atacante suele realizar las siguientes acciones:

- Establece una conexión utilizando protocolos web como HTTP o HTTPS.
- Envía solicitudes (requests) al servidor bajo su control para comprobar que la comunicación está activa.
- Recibe instrucciones o comandos desde el servidor mediante las respuestas HTTP.
- Transmite información del sistema, cuando sea necesario, utilizando el mismo canal de comunicación.
- Mantiene la comunicación activa, mezclando su tráfico con el tráfico web normal para dificultar su detección.

**Herramientas utilizadas:**

- Navegadores web.
- PowerShell.
- CMD.
- Herramientas de administración de red.
- Protocolos HTTP y HTTPS.

**Comandos comunes (Windows):**

Comprobar la conectividad con un servidor:

    ping ejemplo.com

Consultar una página web desde PowerShell:

    Invoke-WebRequest https://ejemplo.com

Mostrar las conexiones de red activas:

    netstat -ano

Verificar la configuración de red:

    ipconfig

Resolver el nombre de un dominio:

    nslookup ejemplo.com

**Nota:** Estos comandos son herramientas legítimas de administración y diagnóstico de Windows. En el contexto de **MITRE ATT&CK**, un atacante puede utilizarlos para comprobar la conectividad de red y establecer comunicaciones mediante protocolos web. La técnica se centra en el uso de HTTP/HTTPS como canal de comunicación, no en los comandos en sí.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Un canal de comunicación con el equipo comprometido utilizando HTTP o HTTPS.
- Capacidad para enviar instrucciones y recibir respuestas del sistema remoto.
- Posibilidad de intercambiar información sin generar un tráfico inusual, ya que utiliza protocolos web comunes.
- Mayor probabilidad de evadir algunos mecanismos de monitoreo al mezclar su comunicación con el tráfico normal de Internet.
- Un medio para continuar con otras etapas del ataque, como el control remoto, la recopilación de información o la exfiltración de datos.

# **TÁCTICA 16: Command and Control**

## T1572: Protocol Tunneling (Túneles de protocolo)

**1. Descripción**
La técnica Protocol Tunneling consiste en encapsular un protocolo de comunicación dentro de otro para ocultar el tráfico entre un equipo comprometido y un servidor remoto. De esta forma, las comunicaciones pueden parecer tráfico normal de la red, como HTTPS, dificultando que los sistemas de monitoreo identifiquen actividades sospechosas.

Además de ocultar la comunicación, esta técnica puede permitir el acceso a servicios que normalmente no serían accesibles desde el exterior, como RDP, SMB u otros protocolos internos.

**2. Ejecución**
Después de obtener acceso a un equipo, el atacante generalmente realiza las siguientes acciones:

- Selecciona un protocolo que utilizará como canal de comunicación (por ejemplo, HTTPS o SSH).
- Crea un túnel para encapsular otro protocolo dentro de esa conexión.
- Envía y recibe información a través del túnel, haciendo que el tráfico parezca legítimo.
- Mantiene la comunicación con el sistema comprometido utilizando el canal creado.
- Aprovecha el túnel para acceder a otros equipos o servicios internos que normalmente no estarían disponibles desde el exterior.

**Herramientas utilizadas:**

- SSH.
- PowerShell.
- OpenSSH.
- Protocolos HTTPS y DNS over HTTPS (DoH).
- Herramientas de administración de red.

**Comandos comunes:**

Verificar las conexiones activas:

    netstat -ano

Comprobar la conectividad con un servidor:

    ping servidor

Crear un túnel SSH (ejemplo de administración remota):

    ssh -L 8080:servidor:80 usuario@equipo_remoto

Comprobar la resolución DNS:

    nslookup ejemplo.com

**Nota:** Estos comandos son herramientas legítimas utilizadas por administradores de sistemas para la gestión remota y el diagnóstico de redes. En el contexto de **MITRE ATT&CK**, la técnica describe cómo un atacante puede aprovechar mecanismos de tunelización para ocultar comunicaciones o acceder a servicios internos, no el uso indebido de estos comandos.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Un canal de comunicación oculto entre el equipo comprometido y un servidor remoto.
- Mayor dificultad para que el tráfico malicioso sea detectado por los sistemas de seguridad.
- Posibilidad de acceder a servicios internos que normalmente no serían accesibles desde Internet.
- Comunicaciones protegidas mediante protocolos cifrados, como HTTPS o SSH.
Un medio para continuar otras etapas del ataque, como el control remoto, el movimiento lateral o la transferencia de información.

# **TÁCTICA 17: Exfiltration**

## T1048.003: Exfiltration Over Alternative Protocol: Exfiltration Over Unencrypted Non-C2 Protocol (Exfiltración a través de un protocolo alternativo: Exfiltración a través de un protocolo no cifrado que no es de comando y control (C2))

**1. Descripción**
La técnica Exfiltration Over Unencrypted Non-C2 Protocol consiste en extraer información de un sistema utilizando protocolos de red que no emplean cifrado, como HTTP, FTP o DNS, y que son diferentes al canal de comando y control (C2) utilizado por el atacante.

Al utilizar protocolos comunes en una red, el tráfico puede parecer una comunicación normal. En algunos casos, los datos se codifican (por ejemplo, con Base64) para facilitar su transmisión, aunque no se encuentran cifrados, por lo que podrían ser leídos si el tráfico es interceptado.

**2. Ejecución**
Después de recopilar la información de interés, el atacante generalmente realiza las siguientes acciones:

- Selecciona los archivos o datos que desea extraer del sistema.
- Prepara la información, organizándola o codificándola para facilitar su envío.
- Elige un protocolo sin cifrado, como HTTP, FTP o DNS.
- Establece una conexión con otro servidor o ubicación de la red.
- Transfiere la información utilizando ese protocolo, procurando que el tráfico se parezca al tráfico normal de la organización.

**Herramientas utilizadas:**

- CMD.
- PowerShell.
- Clientes FTP.
- Herramientas de administración de red.
- Protocolos HTTP, FTP y DNS.

**Comandos comunes (Windows):**

Comprobar la conectividad con un servidor:

    ping servidor

Ver las conexiones de red activas:

    netstat -ano

Resolver un nombre de dominio:

    nslookup ejemplo.com

Codificar un archivo en Base64 mediante PowerShell:

    [Convert]::ToBase64String([IO.File]::ReadAllBytes("archivo.txt"))

**Nota:** Estos comandos son herramientas legítimas de administración y diagnóstico de Windows. En el contexto de MITRE ATT&CK, la técnica describe el uso de protocolos de red sin cifrado para transferir información, no el uso malicioso de estos comandos.

**3. Producto**
Si la técnica se ejecuta con éxito, el atacante obtiene:

- Transferencia de información desde el equipo comprometido hacia otro sistema.
- Uso de protocolos comunes que pueden dificultar la identificación del tráfico como sospechoso.
- Posibilidad de enviar documentos, archivos de configuración u otros datos de interés sin utilizar el canal principal de comando y control (C2).
- Un mecanismo para completar la fase de exfiltración y continuar con el análisis de la información obtenida.
